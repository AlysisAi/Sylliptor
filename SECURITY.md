# Security Policy

## Supported Versions

Sylliptor tracks security support by release line and repository branch, not only by the
current Python package version string. Until the first public beta is cut, package metadata may
still report `0.1.4`; security fixes are handled on the `v0.5` branch / `v0.5.x` line.

| Release line / branch | Supported |
| --------------------- | --------- |
| v0.5.x / `v0.5`       | Yes       |
| Earlier lines         | No        |

## Reporting a Vulnerability

Please report security vulnerabilities by contacting [products@alysisai.com](mailto:products@alysisai.com).

Do NOT open public GitHub issues for security bugs.

We aim to acknowledge security reports within 72 hours and provide a fix or remediation roadmap within 14 days.

Please include:

- Affected version
- Reproduction steps
- Impact assessment

## Disclosure

Coordinated disclosure is preferred. Credit will be given unless the reporter prefers anonymity.
